## 1. プロダクト概要

### 1.1 名称
**gpt-history**

### 1.2 目的
ChatGPT (chatgpt.com) の入力欄で、ターミナル (bash / zsh) のように過去に送信したメッセージを `↑` / `↓` キーで呼び出せるようにする Chrome 拡張機能。

### 1.3 スタンス
- **自分用ツール**として開発する
- GitHub に OSS として公開する
- Chrome Web Store への公開は任意
- メンテナンスは気が向いたときに行う前提
- バズや収益は狙わない

### 1.4 ターゲットユーザー
- 開発者本人
- 同じくターミナル文化に慣れたエンジニアで、興味を持った人

## 2. 機能要件

### 2.1 MVP (最小実装) スコープ

#### F-1: メッセージ送信時の履歴保存
- ChatGPT の入力欄からメッセージが送信されたタイミングで、その内容を履歴に保存する
- 送信検知の手段:
  - 送信ボタンのクリック
  - `Enter` キー (修飾キーなし) による送信
- 保存先: `chrome.storage.local`

#### F-2: 履歴呼び出し (戻る) — `↑`
- 以下の**いずれかの条件**を満たすときに `↑` を奪い、1つ前の履歴を入力欄に挿入する
  - 入力欄が**空**である
  - カーソルが**先頭行**にある (キャレットより上に改行が存在しない)
- 上記条件を満たさないとき (= 複数行入力中で2行目以降にカーソルがある) は、矢印キーを奪わず通常のカーソル移動として動作させる
- さらに `↑` を押すごとに、より古い履歴へ遡る
- 履歴の先頭 (最古) に達したら、それ以上は遡らない (何もしない)

#### F-3: 履歴呼び出し (進む) — `↓`
- 以下の**いずれかの条件**を満たすときに `↓` を奪い、1つ次の履歴を入力欄に挿入する
  - 入力欄が**空**である
  - カーソルが**末尾行**にある (キャレットより下に改行が存在しない)
- 上記条件を満たさないときは、通常のカーソル移動として動作させる
- 履歴の末尾に達したら、入力欄を「履歴呼び出し開始前の状態」に戻す
  - 開始前に何か入力していた場合: その内容に復元
  - 開始前が空だった場合: 空にする

#### F-4: 履歴位置のリセット
- メッセージを送信した時点で、履歴の参照位置をリセット (末尾扱い) する
- 入力欄をユーザーが手動で編集したときは、履歴ナビゲーション状態を解除し、編集を優先する

#### F-5: 履歴の保存ルール
- 保存上限: 直近 1000 件 (超えたら古いものから自動削除)
- 連続する重複は記録しない (bash の `HISTCONTROL=ignoredups` 相当)
- 空文字、空白のみのメッセージは記録しない

### 2.2 V2 候補 (MVP 後に検討)

優先度順:

1. **編集中バッファの保護**
   履歴呼び出し開始時に編集中だった内容を一時退避し、`↓` で最後まで戻ったときに復元する。F-3 の挙動を強化する位置づけ。

2. **`Ctrl+R` インクリメンタル検索**
   bash の `Ctrl+R` 相当。小さな検索 UI を出して履歴を絞り込む。

3. **エクスポート / インポート**
   履歴を JSON でダウンロード / アップロード。マシン移行や事故時のリカバリ用。

4. **オプション画面**
   保存上限、無効化トグルなど。

### 2.3 やらないこと (スコープ外)

- 会話 (スレッド) ごとの履歴管理 → 全体共通の1つの履歴のみ
- 既存の ChatGPT 会話履歴 (DOM 上の過去メッセージ) の取り込み
- ピン留め機能
- クラウド同期 (`chrome.storage.sync` も使わない)
- ChatGPT 以外のサイト対応 (Claude, Gemini など)
- 多言語化

## 3. 非機能要件

### 3.1 対応環境
- **ブラウザ**: Chrome (Manifest V3)
- **副次対応**: Microsoft Edge (Chrome 拡張がそのまま動く想定、明示的なテストはしない)
- **非対応**: Firefox, Safari
- **対象 URL**: `https://chatgpt.com/*`

### 3.2 パフォーマンス
- 履歴呼び出しはキー押下から 100ms 以内に反映
- ChatGPT の通常操作 (タイピング、送信) に体感できる遅延を加えない

### 3.3 保守性
- ChatGPT の DOM 変更で壊れることは織り込み済み
- 入力欄や送信ボタンの取得ロジックは1ファイルに集約し、壊れたときの修正箇所を限定する
- 壊れたら直す、直せなければ放置する程度の運用で OK

### 3.4 プライバシー
- 履歴は完全にローカル (`chrome.storage.local`) のみ
- 外部送信は一切行わない
- README に明記する

## 4. キーバインド仕様

| キー | 動作 | 発火条件 |
|---|---|---|
| `↑` | 1つ前の履歴を呼び出す | 入力欄が空、または**カーソルが先頭行にある** |
| `↓` | 1つ次の履歴 (または編集前の状態) を呼び出す | 入力欄が空、または**カーソルが末尾行にある** |
| `↑` / `↓` (発火条件を満たさない) | **奪わない** (通常のカーソル移動) | カーソルが2行目以降〜末尾行未満 |

### 4.1 「先頭行」「末尾行」の判定方法
- 入力欄は ChatGPT の実装に応じて `<textarea>` または `contenteditable` の `<div>`
- **先頭行**: キャレット位置より前のテキストに改行 (`\n` または `<br>`) が含まれない
- **末尾行**: キャレット位置より後のテキストに改行が含まれない
- 1行しか入力していない場合は、先頭行かつ末尾行として扱う

これにより bash / zsh と同じ感覚で履歴ナビゲーションができる。

## 5. 技術スタック

- **Manifest V3** (`manifest.json`)
- **Content Script** で ChatGPT のページに介入
- **Storage**: `chrome.storage.local`
- **ビルド**: 最初は素の JS で良い。必要に応じて TypeScript + Vite を検討
- **言語**: JavaScript (または TypeScript)

## 7. リスクと割り切り

| リスク | 対応 |
|---|---|
| ChatGPT の DOM 変更で動かなくなる | `dom.js` に集約。気が向いたら直す |
| OpenAI 公式が同等機能を実装する | そうなったら拡張を引退させる。それで OK |
| 拡張機能が誰にも使われない | 自分が使えれば成功。発見性は気にしない |
| 矢印キー単体奪取で他のショートカットと衝突 | 発火条件 (先頭行/末尾行 + 空欄) で厳密に絞ることで誤爆を防ぐ |
