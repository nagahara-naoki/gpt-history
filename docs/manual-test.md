# 手動テスト チェックリスト

本拡張機能は自動テストを持たないため、リリース前に以下を手動で確認します。

## 準備

- [ ] `pnpm install` で依存をインストール
- [ ] `pnpm build` を実行して `dist/` を生成
- [ ] Chrome の `chrome://extensions/` で `dist/` を読み込む
- [ ] https://chatgpt.com を開く (ログイン済みであること)

## 基本動作

### F-1: 開いているチャット履歴の取得

- [ ] 入力欄に `hello` と入力し、Enter で送信 → 現在のチャット内履歴として呼び出せる
- [ ] 入力欄に `world` と入力し、送信ボタンクリックで送信 → 現在のチャット内履歴として呼び出せる
- [ ] 既存チャットを途中から開き、入力欄で `↑` を押す → そのチャットに表示されている自分の直近発言が呼び出される
- [ ] 別チャットを開いた直後に `↑` を押す → 前のチャットの履歴は表示されない

### F-2: `↑` で履歴を遡る

- [ ] 入力欄が空のときに `↑` を押す → 直前に送信したメッセージが入力欄に挿入される
- [ ] 続けて `↑` を押す → さらに古い履歴に切り替わる
- [ ] 最古の履歴に達したあと、もう一度 `↑` を押す → 何も起こらない (それ以上遡らない)

### F-3: `↓` で履歴を進める

- [ ] `↑` を数回押した状態で `↓` を押す → 1 つ新しい履歴に切り替わる
- [ ] 末尾を超えて `↓` を押す → ナビゲーション開始前の入力内容に戻る
  - 開始前に何か入力していた場合: その内容に復元される
  - 開始前が空だった場合: 空になる

### F-4: 履歴位置のリセット

- [ ] `↑` で履歴を呼び出した状態で送信 → 送信後、`↑` でまた最新履歴から始まる
- [ ] `↑` で履歴を呼び出した状態で入力欄に文字を追加 → ナビゲーション状態が解除され、その後 `↑` を押すと最新履歴から始まる

### F-5: 履歴候補ルール

- [ ] 同じメッセージを 2 回連続で送信 → 履歴候補では 1 件として扱われる (連続重複排除)
- [ ] 空文字または空白のみのメッセージを送信しようとする → 履歴候補に入らない

## 行判定 (重要)

### 単一行入力

- [ ] 1 行だけ入力した状態で `↑` を押す → 履歴が呼び出される (先頭行扱い)
- [ ] 1 行だけ入力した状態で `↓` を押す → 履歴ナビゲーション中なら次、未開始なら通常動作 (末尾行扱い)

### 複数行入力 (Shift+Enter で改行)

- [ ] 2 行入力し、カーソルが 1 行目にある状態で `↑` を押す → 履歴が呼び出される (先頭行)
- [ ] 2 行入力し、カーソルが 2 行目にある状態で `↑` を押す → 通常のカーソル移動。履歴は呼ばれない
- [ ] 3 行入力し、カーソルが 2 行目にある状態で `↑` を押す → 通常のカーソル移動 (1 行目へ)
- [ ] 3 行入力し、カーソルが 2 行目にある状態で `↓` を押す → 通常のカーソル移動 (3 行目へ)
- [ ] 3 行入力し、カーソルが 3 行目 (末尾行) にある状態で `↓` を押す → 履歴ナビゲーション中なら次、未開始なら通常動作

## エッジケース

### IME (日本語入力)

- [ ] 「あ」を入力中 (変換確定前) に `↑` を押す → 履歴は呼ばれず、IME 候補ウィンドウの操作になる
- [ ] 「あ」を入力中に `Enter` を押す → 送信されず、変換確定として動作する

### 修飾キー

- [ ] `Shift+↑` を押す → 履歴は呼ばれず、通常の選択拡張になる
- [ ] `Ctrl+↑` を押す → 履歴は呼ばれない
- [ ] `Shift+Enter` を押す → 送信されず、改行になる

### 非永続化 / チャット切替

- [ ] いくつか送信してからブラウザを再起動 → 開いているチャットに表示されている発言だけが履歴候補になる
- [ ] DevTools → Application → Extension storage に `history` キーが作られていない
- [ ] チャット A で `↑` が使えることを確認後、チャット B を開く → チャット A の発言は表示されない

## 互換性 / 非破壊性

- [ ] ChatGPT 本体の通常操作 (タイピング、送信、新規チャット作成) が阻害されていない
- [ ] 拡張機能を無効化すると、↑↓ が完全に通常動作に戻る
- [ ] エラーが Console に出ていない (出ていても warn 以下で、機能が無効化されるだけになっている)

## DOM 変更耐性

- [ ] ChatGPT の入力欄が見つからない状況 (例: ログアウト画面) で、Console に warn は出るがエラーは出ない
