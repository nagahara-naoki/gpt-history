## 2. コード品質方針

### 2.1 前提
本プロジェクトは OSS として GitHub に公開する。他者がコードを読む、Issue や PR を送ることを想定し、以下を満たす。

### 2.2 可読性のルール
- **1ファイル 1責任** を守る (例: `caret.js` はキャレット判定のみ)
- **関数は20行を目安**に分割する
- **マジックナンバー禁止**。定数は `constants.js` または各モジュール上部にまとめる
- **早期 return** を活用し、ネストを浅く保つ
- **副作用のある関数**と**純粋関数**を意識的に分ける
- 命名は意味が伝わるものにする (`tmp`, `data`, `x` などの曖昧な名前は避ける)

### 2.3 コメントの書き方
- **「何を」ではなく「なぜ」を書く**
  - ❌ `// i に 1 を足す`
  - ✅ `// ChatGPT の DOM 更新が次フレームで起きるため、1 つずらして参照する`
- **モジュール冒頭にファイルの責務をコメントで明記**する

  ```js
  // caret.js
  // textarea / contenteditable のキャレット位置を取得し、
  // 「先頭行か」「末尾行か」を判定する責務を持つ。
  ```

- **関数には JSDoc を付ける**
  ```js
  /**
   * キャレットが入力欄の先頭行にあるかを判定する。
   *
   * @param {HTMLElement} el - 入力欄要素 (textarea または contenteditable)
   * @returns {boolean} 先頭行なら true
   */
  function isAtFirstLine(el) { ... }
  ```

### 2.4 エラーハンドリング
- ChatGPT の DOM が変わって要素が見つからない場合は、`console.warn` で警告を出し、機能を**サイレントに無効化**する (拡張機能が ChatGPT 本体の動作を壊さないことを最優先)
- `try / catch` で囲むのは外部 API (`chrome.storage` など) に触れる箇所のみ

## 3. 技術スタック

### 3.1 ビルドツール: Vite
- `vite` を使ってビルドする
- Chrome 拡張機能向けに `@crxjs/vite-plugin` を使用する (`manifest.json` の HMR・自動再ビルドが可能)
- 出力先: `dist/`
- 開発用コマンド: `npm run dev` (HMR 有効)
- ビルドコマンド: `npm run build` (`dist/` に成果物を出力)

### 3.2 言語: JavaScript (TypeScript 不採用)
- MVP 段階では **素の JavaScript (ES Modules)** を採用
- 理由: 個人用ツールでスコープが小さく、型を入れるコストがメリットを上回らない
- 将来コードベースが大きくなったら TypeScript への移行を検討する (`V2` 候補)

### 3.3 リンター / フォーマッター: Biome
- **Biome** を採用 (ESLint + Prettier の代替、単一ツールで完結)
- 設定ファイル: `biome.json`
- フォーマット: シングルクォート、セミコロンあり、インデント 2 スペース、行幅 100
- リンター: 推奨ルールを基本に有効化
- コマンド:
  - `npm run lint`: チェックのみ
  - `npm run format`: 自動整形
  - `npm run check`: lint + format のチェック

### 3.4 Git hooks (任意)
- `lefthook` または `simple-git-hooks` を使い、コミット前に `biome check` を走らせる
- MVP では必須ではないが、可能なら入れる

### 3.5 ストレージ
- `chrome.storage.local` のみ
- ラッパー (`src/storage.js`) で薄く包む
- スキーマ:
  ```js
  {
    "history": ["msg1", "msg2", ...],  // 古い順
    "schemaVersion": 1                   // 将来マイグレーション用
  }
  ```

### 3.6 Manifest V3

## 5. ポリシーファイル

OSS として、また将来 Chrome Web Store に公開する可能性も視野に、以下のファイルを用意する。

### 5.1 LICENSE
- **MIT License** を採用
- 理由: シンプルで制約が少なく、OSS で最も一般的

### 5.2 PRIVACY_POLICY.md (プライバシーポリシー)
Chrome Web Store 公開時には**必須**。以下を明記する。

- 本拡張機能が**収集する情報**: ユーザーが ChatGPT に送信したメッセージのテキストのみ
- **保存場所**: ユーザーのローカル PC 内 (`chrome.storage.local`)
- **外部送信の有無**: なし。一切外部に送信しない
- **第三者提供**: なし
- **アナリティクスの有無**: なし
- **問い合わせ先**: GitHub Issues へのリンク
- **改訂日**

### 5.3 README.md (重要)
拡張機能のインストール率と安心感を高めるため、以下のセクションを必ず含める。

1. **タイトルと一行説明**
2. **デモ GIF / スクリーンショット** (`docs/screenshots/` に配置)
3. **特徴 (Features)**
   - 矢印キーだけで履歴を呼び出せる (修飾キー不要)
   - 完全ローカル保存、外部通信なし
   - 複数行入力中のカーソル移動を壊さない (先頭行/末尾行判定)
4. **インストール方法**
   - (A) GitHub からダウンロードして「パッケージ化されていない拡張機能を読み込む」手順
     - 画面付きでステップごとに記載
   - (B) Chrome Web Store からインストール (公開した場合)
5. **使い方**
   - `↑` で前の履歴、`↓` で次の履歴
   - 発火条件 (空欄 or 先頭行/末尾行) の説明
6. **プライバシー** (1ブロック設けて強調)
   - 「**履歴はあなたのブラウザ内にのみ保存され、外部に送信されることは一切ありません**」
   - `PRIVACY_POLICY.md` へのリンク
7. **既知の制限**
   - ChatGPT の UI 変更で動かなくなる可能性
   - 一部の入力モード (IME 変換中など) では発火しないことがある
8. **開発者向け (Development)**

9. **コントリビューション**

## 7. 開発ワークフロー

### 7.1 セットアップ手順
```bash
git clone https://github.com/<user>/gpt-history.git
cd gpt-history
npm install
npm run dev
```

### 7.2 Chrome への読み込み (開発時)
1. Chrome で `chrome://extensions/` を開く
2. 「デベロッパーモード」を ON
3. 「パッケージ化されていない拡張機能を読み込む」で `dist/` を選択

### 7.3 リリース手順
1. `CHANGELOG.md` を更新
2. `package.json` と `manifest.json` のバージョンを上げる
3. `npm run build` で `dist/` を生成
4. `dist/` を zip 化
5. GitHub の Releases にアップロード

## 8. テスト方針

- MVP では**自動テストは入れない** (個人用ツールにつきコストが見合わない)
- 手動テスト項目を `docs/manual-test.md` にチェックリストとして用意する
  - 1行入力で `↑` → 履歴呼び出し
  - 複数行入力で2行目から `↑` → 通常のカーソル移動
  - 履歴の先頭/末尾での挙動
  - メッセージ送信後の履歴位置リセット
  - ブラウザ再起動後の永続化

## 9. 実装の順序 (推奨)

依存関係と動作確認のしやすさを考慮した推奨順序:

1. プロジェクト初期化 (`package.json`, Vite, Biome, `manifest.json`)
2. `storage.js` (履歴の読み書き) を実装、コンソールで動作確認
3. `dom.js` (入力欄取得) を実装、ChatGPT 上で要素が取れるか確認
4. `caret.js` (先頭行/末尾行判定) を実装、ChatGPT 上で動作確認
5. `keybind.js` + `history.js` を実装、キーバインドで履歴ナビゲーションが動くか確認
6. `content.js` で全体を組み立て
7. アイコン作成
8. README / プライバシーポリシー等のドキュメント整備
9. GitHub に公開
